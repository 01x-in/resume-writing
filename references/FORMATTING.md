# Resume Formatting Guide

## Fonts, layout, ATS compliance, and formatting rules.

---

## Output Format

Always produce a **clean text-only PDF**. Create in Word/Docs/Markdown, export as PDF for submission. PDF preserves formatting across all devices and ATS systems.

---

## Fonts

Use professional, modern, ATS-friendly fonts:

| Font | Notes |
|------|-------|
| **Aptos** | Microsoft's newest default. Modern, clean. First choice. |
| **Calibri** | Microsoft standard. Universally supported. Safe choice. |
| **Arial** | Classic fallback. Works everywhere. |

**Do NOT use:** Times New Roman (dated), Comic Sans (never), decorative fonts, or fonts requiring installation. If unsure, use Calibri.

### Font Sizes

| Element | Size |
|---------|------|
| Name | 18-22pt |
| Section headings | 14-16pt |
| Body text | 10-12pt (never smaller than 10pt) |
| Contact info | 10-11pt |

---

## Page Layout

### Margins
- **Standard:** 1 inch (72pt) on all sides
- **Tight (more content):** 0.75 inch (54pt) on all sides
- **Minimum:** 0.5 inch — never go smaller

### Column Layout
- **Single column ONLY.** Multi-column layouts confuse ATS systems and are harder to scan.
- No sidebars, no two-column headers, no floating elements.

### Page Count
- **Mid-to-senior level (5+ years):** Maximum 2 pages
- **Entry level (<5 years):** 1 page
- **Executive/Director (15+ years):** 2 pages still preferred, 3 maximum

---

## Formatting Do's

- Use consistent heading hierarchy (H1 for name, H2 for sections, H3 for role titles)
- Use bullet points for experience (not paragraphs)
- Use bold for role titles and company names
- Use consistent date format throughout (either "Jan 2022 – Present" or "01/2022 – Present", not both)
- Use horizontal rules or subtle borders to separate sections
- Left-align all text (no centered body text)
- Dates right-aligned on the same line as role titles (if the format supports it)

## Formatting Don'ts

- **No tables.** Many ATS systems cannot parse tables correctly.
- **No multi-column layouts.** ATS reads left-to-right, top-to-bottom — columns scramble the order.
- **No graphics, icons, or images.** ATS cannot read them. Recruiters don't need them.
- **No photos.** Not required in U.S./U.K. and can trigger bias.
- **No Canva templates.** They look flashy but fail ATS parsing due to hidden tables and graphics.
- **No headers/footers with critical information.** Some ATS systems skip headers/footers entirely. Put your name and contact info in the body.
- **No text boxes or floating elements.** ATS parses document flow, not positioned elements.
- **No color as the only indicator.** If a section heading is only distinguished by color (no size or weight change), it may be invisible to ATS and in black-and-white prints.

---

## ATS Compliance Checklist

Applicant Tracking Systems parse resumes before humans see them. These rules ensure your resume survives ATS parsing:

- [ ] Single-column layout (no tables, no multi-column)
- [ ] Standard section headings: "Work Experience" not "My Journey" — ATS looks for standard labels
- [ ] Text is selectable (not an image of text)
- [ ] File format: PDF (preferred) or DOCX (if specifically requested)
- [ ] No text in headers/footers (some ATS skip these)
- [ ] Standard fonts (Aptos, Calibri, Arial)
- [ ] No special characters in section headings
- [ ] Contact info in the body, not in a text box
- [ ] File name: `Firstname_Lastname_Resume.pdf` (professional, parseable)

---

## White Space & Density

- **Don't cram.** If you need to shrink font below 10pt to fit content, cut content instead.
- **Don't waste.** Every line should earn its place. If a bullet doesn't show impact, remove it.
- **Section spacing:** 12-16pt between sections, 4-8pt between bullets.
- **Paragraph spacing in summary:** 1.2-1.4 line height for readability.
- **Bullet indent:** 0.25-0.5 inch from left margin.

---

## Print Test

Before submitting, verify the resume looks good when:
1. Printed in black and white (no information lost)
2. Printed on standard letter paper (US) or A4 (international)
3. Viewed on a phone screen (many recruiters do initial screening on mobile)
4. Copy-pasted into plain text (simulates ATS extraction — check that content order makes sense)
